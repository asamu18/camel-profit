**原因分析：**
主页（Dashboard）的引导框位置“偏低”，最根本的原因是 **CSS 定位上下文（Positioning Context）** 的问题。
虽然引导组件使用了 `fixed`（固定定位），但它当前是渲染在 `Dashboard` 组件内部的。在某些复杂的布局中（特别是涉及 `transform`、`filter` 或特定的 flex/grid 嵌套时），`fixed` 定位可能会意外地“降级”，不再相对于整个浏览器视口定位，而是相对于最近的某个父级容器定位。
如果这种情况发生，引导框的坐标计算（它是基于视口绝对坐标的）就会叠加父级容器的偏移量（例如顶部的 Header 高度），导致视觉上整体下移，也就是你看到的“偏低”。历史页面可能因为内容结构不同，没有触发这个特定的 CSS 边界情况，所以显示正常。

**解决方案：**
使用 Vue 3 的 **`<Teleport>`** 功能。
这是一个“传送门”技术，它可以强制将引导组件的 DOM 节点直接挂载到网页的 `<body>` 根节点下，而不是留在 `Dashboard` 组件内部。
这样做可以彻底脱离任何父级组件的样式干扰，确保 `fixed` 定位永远是相对于浏览器窗口的，从而保证定位 100% 精准。

**实施计划：**
1.  修改 `src/components/UserGuide.vue`，用 `<Teleport to="body">` 包裹整个模态框结构。
2.  这不仅能修复主页的偏移问题，也能预防未来其他页面出现类似的层级遮挡或定位错误。
