## 数据层

### Supabase Client

- [supabase.js](file:///workspace/src/lib/supabase.js)
  - 读取 `VITE_SUPABASE_URL` 与 `VITE_SUPABASE_KEY`
  - 根据环境决定 `supabaseUrl`：生产 Web 使用代理路径 `/api/supabase`，其他情况直连真实 URL
  - `createClient(supabaseUrl, supabaseKey)` 导出 `supabase`

### 数据访问聚合层（dataService）

- [dataService.js](file:///workspace/src/api/dataService.js)
  - 目标：把常用 CRUD 与复合查询封装为可复用 API，降低组件中 Supabase 查询散落的复杂度
  - 覆盖的表：`income` / `cost` / `settings` / `inventory` / `posts`

关键方法清单（按业务归类）：

- 用户上下文
  - `_getUserId()`：通过 `supabase.auth.getUser()` 获取当前用户 id
- 收入/支出 CRUD
  - `addIncome(record)` / `addCost(record)`：自动带 `user_id` 插入
  - `updateRecord(table, id, updates)`：按 id 更新
  - `deleteRecord(table, id)`：按 id 删除
  - `clearAllUserData()`：清空当前用户所有表数据并清本地缓存（高风险操作）
- 查询
  - `getDataByMonth(monthStr)`：按月份查询 income/cost 并排序
  - `getTrendData()`：近 6 个月趋势数据（date+amount）
  - `getFullHistory({ page, pageSize, type, startDate, endDate })`：合并 income/cost 并分页切片
- 库存（inventory）
  - `getInventory()`：读取当前用户库存列表
  - `updateInventory(record)`：按 `user_id, category` upsert
  - `incrementInventory(record)`：读取旧 quantity 并累加 upsert（用于进货自动累加）

## 业务页面与组件

### 页面：Dashboard（首页）

- [Dashboard.vue](file:///workspace/src/components/Dashboard.vue)
  - 负责：显示“今日净赚 / 本月利润预估 / 固定成本”等核心数据卡片，并作为记账操作入口
  - 数据来源：直接用 `supabase.from(...).select(...)` 拉取 income/cost/settings（见 [Dashboard.vue](file:///workspace/src/components/Dashboard.vue#L240-L255)）
  - 关键计算：
    - `dailyPotentialIncome`：基于 settings（交奶频率/单价/每次公斤）估算日均收入
    - `dailyFixedCost`：基于 settings.daily_template 计算日固定成本
    - `todayProfit`：已交奶则按 `todayIncome / duration` 平摊，否则用日均预估收入
    - `monthlyProfit`：当月实际交奶覆盖天数 + 剩余天数按预估收入补齐，再减去固定成本与杂费
  - 关键副作用：
    - `autoFillMissingCosts(userId, dailyTemplate)`：根据最后一条“日常支出”日期，向后补齐直到今天（保证利润计算连续）
    - watch `route.query.action === 'bulkImport'`：打开 [ImportMilkModal.vue](file:///workspace/src/components/ImportMilkModal.vue)
  - 弹窗：AddRecordModal / SettingsModal / SetupWizard / ImportMilkModal / UserGuide（引导）

### 页面：HistoryView（历史账本）

- [HistoryView.vue](file:///workspace/src/components/HistoryView.vue)
  - 负责：按时间范围与类型（全部/收入/支出/饲料）查看汇总与明细；支持删除；饲料视图显示库存估值与“可用天数”
  - 数据来源：
    - income/cost 直接查询
    - inventory 通过 `dataService.getInventory()`（见 [HistoryView.vue](file:///workspace/src/components/HistoryView.vue#L277-L285)）
    - settings 用于 daily_template 推算库存可用天数
  - 汇总 stats：
    - `milkIncome` / `camelIncome`：按 category 简单分类
    - `dailyCost` / `extraCost` / `feedCost`：按 `cost_type` 统计（`库存进货`、`日常支出`、其他）
  - 支出视图的“本月累计”：
    - 对每月支出按 category 聚合为 “本月累计”条目，并可展开查看 children（见 [HistoryView.vue](file:///workspace/src/components/HistoryView.vue#L296-L320)）

### 组件：Login（登录/注册）

- [Login.vue](file:///workspace/src/components/Login.vue)
  - 方案：手机号 + 密码，但因为 Supabase 需要 email 格式，拼接固定后缀 `@camel.local`（见 [Login.vue](file:///workspace/src/components/Login.vue#L52-L66)）
  - 流程：先 `signInWithPassword`；若是 “Invalid login credentials” 则 `signUp` 再尝试登录

### 组件：AddRecordModal（核心记账弹窗）

- [AddRecordModal.vue](file:///workspace/src/components/AddRecordModal.vue)
  - 统一的“多场景”弹窗，通过 `openWithScene(scene)` 切换：
    - `卖奶`：录入交奶（防重复：同日同 category 只允许一条，见 [AddRecordModal.vue](file:///workspace/src/components/AddRecordModal.vue#L212-L216)）
    - `买饲料`：记录进货支出 + 自动 `incrementInventory` 累加库存
    - `录入库存`：盘点库存（支持按吨或按袋换算）
    - `其他`：杂费/额外收入（通过 `form.type` 区分）

### 组件：ImportMilkModal（批量导入交奶记录）

- [ImportMilkModal.vue](file:///workspace/src/components/ImportMilkModal.vue)
  - 解析粘贴文本：自动识别日期、重量、单价，生成 `parsedRecords`
  - 时长分配：根据跨度与条数给每条记录分配 `duration`（或单条时使用 settings.milk_frequency）
  - 提交：批量插入 income，然后按跨度补齐缺失的“日常支出”（见 [ImportMilkModal.vue](file:///workspace/src/components/ImportMilkModal.vue#L161-L175)）

### 组件：SettingsModal（经营设置）

- [SettingsModal.vue](file:///workspace/src/components/SettingsModal.vue)
  - 维护 settings 表字段：规模、交奶预估参数、每日固定成本模板
  - `upsert({ user_id, ...form })` 全量覆盖，确保模板/奶价等同步更新（见 [SettingsModal.vue](file:///workspace/src/components/SettingsModal.vue#L123-L135)）
  - 提供：重新观看引导、清空所有数据、退出账号

### 组件：SetupWizard（首次初始化）

- [SetupWizard.vue](file:///workspace/src/components/SetupWizard.vue)
  - `check()`：若 settings.daily_template 为空，则弹出初始化向导
  - `saveSettings()`：写入 settings；同时插入当日 cost（日常支出模板）与 income（可选）并设置 `localStorage.is_new_user`
